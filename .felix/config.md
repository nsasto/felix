# Felix Agent Configuration

This file documents the configuration options in `felix/config.json` for customizing the Felix agent's behavior, particularly the LLM agent integration.

> **Note (Phase -1)**: As of v0.1-cleanup-complete, **felix/state.json** is no longer read by the backend for runtime state management. The backend endpoints for agent registry and status are stubbed pending the Phase 0 database implementation. The state.json file is still written by felix-agent.ps1 for local agent execution, but the backend does not consume it.

## How Agent Selection Works

- `felix/config.json` selects a local CLI agent key via `agent.agent_id`
- Local CLI presets (optional overrides) live in **.felix/agents.json** in the repo
- Felix CLI configuration and execution are always local to the machine running it.
- Backend agent profiles are stored in the database and managed via the API

```json
{
  "agent": {
    "agent_id": "ag_123456789"
  }
}
```

Legacy numeric IDs are still accepted for compatibility and are migrated by setup when a key-based profile is selected.

When `felix setup` finds exactly one configured profile, it auto-selects that profile as active and skips the manual chooser.

## Local CLI Presets (**.felix/agents.json**)

Each preset is an entry under `agents`:

```json
{
  "agents": [
    {
      "id": "ag_123456789",
      "name": "codex",
      "adapter": "codex",
      "model": "gpt-5.2-codex",
      "working_directory": ".",
      "environment": {}
    }
  ]
}
```

### Fields

- `id`: Agent key (content-addressed) generated by setup
- `name`: Display name
- `adapter`: Adapter type (`droid`, `claude`, `codex`, `gemini`)
- `provider`: Optional provider override (defaults to adapter or name)
- `model`: Optional model override (defaults to adapter default)
- `executable`: Optional CLI command override (defaults to adapter default)
- `working_directory`: Optional working dir (relative to project root)
- `environment`: Optional env vars for the agent process

### Defaults vs Overrides

Agent adapters provide defaults for `executable`, `model`, `working_directory`, and `environment`.
Entries in **.felix/agents.json** override those defaults per project.

### OpenAI Codex CLI

Install:

```json
{
  "id": "ag_123456789",

  "adapter": "codex",
  "model": "gpt-5.2-codex",

}
```

## Troubleshooting

- **Agent not found**: Verify the `executable` path exists and is accessible
- **Authentication errors**: Check `environment` variables for required API keys
- **Command failures**: Test the agent manually by piping a prompt into the executable

For more information, see the Felix documentation in the project root.
