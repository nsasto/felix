# S-0032: Remove Backend File Operations

**Phase:** -1 (Legacy Code Cleanup)  
**Effort:** 6-8 hours  
**Priority:** Critical  
**Dependencies:** S-0031

---

## Narrative

This specification covers removing all backend code that directly reads or writes to `felix/state.json`, `felix/requirements.json`, and `felix/agents.json`. This includes cleaning up file-based state management across multiple routers and utility modules.

The goal is to eliminate approximately 800-900 lines of file I/O code and prepare the backend for database-driven state management in Phase 0.

---
